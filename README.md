# ⚾ Baseball Game (Number Baseball) - Unreal Engine 5

언리얼 엔진 5를 활용하여 C++로 구현한 3자리 숫자 야구 게임 프로젝트입니다. 서버-클라이언트 구조에서의 판정 로직 구현과 데이터 동기화를 학습하기 위한 과제입니다.

---

###  과제 소개
본 프로젝트는 고전적인 **숫자 야구 게임**을 언리얼 엔진의 C++ 환경에서 구현하는 것을 목표로 합니다.

* **게임 규칙:**
    * 서버가 중복되지 않는 3자리의 난수를 생성합니다.
    * 플레이어가 3자리의 숫자를 입력하여 정답을 추측합니다.
    * **Strike:** 숫자와 자릿수가 모두 일치하는 경우
    * **Ball:** 숫자는 일치하지만 자릿수가 다른 경우
    * **Out:** 일치하는 숫자가 하나도 없는 경우
* **핵심 목표:**  `GameModeBase`에서 서버 사이드 판정 로직 구현.
    * RPC(Remote Procedure Call)를 통한 서버-클라이언트 통신 이해.
    * `PlayerState`를 활용한 플레이어별 시도 횟수 관리.

---

###  프로젝트 개요
* **개발 환경:** Unreal Engine 5.5 / Visual Studio 2022
* **개발 언어:** C++
* **주요 기능:**
    * 서버 기반의 숫자 야구 판정 로직 구현
    * 클라이언트 입력값 검증 및 결과 멀티캐스트 알림
    * 게임 상태 관리 및 플레이어 접속 처리

---

###  주요 구현 내용

#### 1. GameModeBase (판정 및 흐름 제어)
* `OnPostLogin`을 통한 플레이어 접속 관리 및 데이터 초기화.
* 스트라이크(Strike)와 볼(Ball)을 판정하는 핵심 알고리즘 구현.
* 게임의 승리 조건(3 Strike) 달성 시 전체 클라이언트 알림 처리.

#### 2. PlayerController & UI 연동
* 클라이언트의 입력값을 서버로 전달하는 `Server_SubmitGuess` RPC 구현.
* 입력 데이터 유효성 검사(3자리 숫자, 중복 여부 등) 로직 포함.
* 서버의 판정 결과를 모든 플레이어 화면에 출력하는 `Multicast_DisplayResult` 구현.

###  기술적 도전 및 해결 과정 (Troubleshooting)

과제 진행 중 다음과 같은 기술적 난관에 부딪혔으며, 이를 해결하기 위한 분석 과정을 거쳤습니다.

#### 1. UE 5.5 환경에서의 PlayerState 빌드 이슈
* **문제 상황:** `PlayerState` 상속 클래스 추가 시 엔진 헤더(`PlayerState.h`)에서 대량의 구문 오류 발생.
* **원인:**  가상 메모리(Paging File) 부족으로 빌드 도구(UBT)의 파일 매핑 실패(`MapViewOfFile`).
* **대응 방안:** `Intermediate` 폴더 초기화 및 가상 메모리 확보를 통해 환경 복구 시도. 현재 환경 이슈로 기능을 보류했으나, 원인을 파악하여 추후 안정적인 개발 환경 구축을 위한 기반을 마련할 예정.

###  클래스 구조 (Class Architecture)

| 폴더 | 클래스명 | 역할 및 주요 기능 |
| :--- | :--- | :--- |
| **Game** | `ABGGameModeBase` | 숫자 야구 판정 알고리즘 및 게임 흐름 제어 |
| **Game** | `ABGGameStateBase` | 게임 전역 상태 및 점수 데이터 동기화 관리 |
| **Player** | `ABGPlayerController` | 사용자 입력 처리 및 서버-클라이언트 RPC 담당 |
| **Player** | `ABGPawn` | 플레이어의 기본 물리적 표현 및 조종 객체 |
| **Player** | `ABGPlayerState` | (예정) 플레이어 개별 데이터(시도 횟수 등) 관리 |
| **UI** | `UNBWidget` | 숫자 입력 및 판정 결과(1S1B 등) 표시 UI |
